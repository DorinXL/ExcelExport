# Excel2Json 导表工具使用说明

## 1. 工具用途

本工具用于将 Excel 配置表一键导出为：

- Unity 使用的 JSON 配置文件
- 程序使用的 C# 配置类文件
- 配置清单 `manifest.json`
- 配置表枚举 `ConfigTables.cs`

适用对象：策划、配置同学。

运行环境：Windows。

---

## 2. 你会拿到什么

发布包中至少包含以下文件：

```text
v0.2.0/
├── Excel2Json.exe
├── gen_data.bat
├── 策划使用说明.md
├── CHANGELOG.md
└── Excel/
```

说明：

- `Excel2Json.exe`：导表程序
- `gen_data.bat`：启动脚本，推荐双击它运行
- `策划使用说明.md`：本说明文档
- `CHANGELOG.md`：版本更新日志
- `Excel`：放置待导出的配置表

---

## 3. 最推荐的使用方式

请按下面的目录结构摆放文件：

```text
导表工具目录/
├── Excel2Json.exe
├── gen_data.bat
├── 策划使用说明.md
└── Excel/
    ├── Item.xlsx
    ├── Skill.xlsx
    ├── Store.xlsx
    └── 其他配置表.xlsx
```

注意：

- 必须在 `Excel2Json.exe` 同级目录下新建 `Excel` 文件夹
- 所有要导出的 `.xlsx` 文件都放进 `Excel` 文件夹
- 工具会递归扫描 `Excel` 目录中的所有 `.xlsx` 文件
- 以 `~` 开头的临时文件会自动忽略

---

## 4. 一键导表步骤

### 第一步：准备 Excel

- 把所有配置表放进 `Excel` 文件夹
- 确保 Excel 文件扩展名为 `.xlsx`
- 关闭正在编辑的 Excel 文件后再导表，避免生成临时文件或文件占用

### 第二步：运行工具

推荐方式：

- 直接双击 `gen_data.bat`

也可以：

- 直接双击 `Excel2Json.exe`

### 第三步：查看结果

如果导表成功，会在工具同级目录生成 `Output` 文件夹。

目录结构如下：

```text
Output/
├── Json/
│   ├── Item.json
│   ├── Skill.json
│   └── ...
├── Scripts/
│   ├── ItemConfig.cs
│   ├── SkillConfig.cs
│   └── ...
├── manifest.json
└── ConfigTables.cs
```

## 5. Excel 填表规范

每个需要导出的 Sheet 必须遵循以下格式（V3 新规）：

| 行号 | 内容 | 说明 |
|------|------|------|
| 第 1 行 | 名称 | 表名描述（工具不解析，可写任意内容） |
| 第 2 行 | 类型 | 填写类型标记 |
| 第 3 行 | 默认值 | 若某列配置了默认值，则下方留空数据行将使用该值。如果默认值留空，且数据行也留空，则使用系统内置默认值（数值型为 0，布尔为 false，字符串为空串，数组为空数组） |
| 第 4 行 | 英文标识 | 对应 JSON 和 C# 里的字段名 |
| 第 5 行 | 中文标识 | 对应中文注释，导表时跳过 |
| 第 6 行起 | 数据 | 实际配置内容 |

示例：

```text
道具表       (空)        (空)        (空)
int       string      float     array_int
          (空)        0         (空)
id        name        attack    skill_ids
主键ID    道具名称    攻击力    可用技能列表
1001      铁剑        15.5      [101, 102]
1002      木盾                  [201]
#         被注释行    0         0
```

> **注意：**
> 1. 上例中“木盾”的攻击力留空，由于第 3 行默认值填了 `0`，导出时该值将自动被解析为 `0`。如果某列第 3 行也为空（例如“可用技能列表”），数据也为空，则会使用系统内置默认值（空数组）。
> 2. 在第 6 行起的数据行中，如果第一列（比如主键列）填入了以 `#` 开头的内容，则工具会忽略整行，不进行导出。

### 字段命名要求

- 每个 Sheet 必须包含 `id` 字段
- `id` 字段类型必须是 `int`
- 第 2 行字段名会直接用于程序生成，不会自动改名
- 字段名建议只使用英文字母、数字、下划线

### 支持的数据类型

| 类型标记 | 说明 | 示例 |
|----------|------|------|
| `int` | 整数 | `1001` |
| `float` | 浮点数 | `15.5` |
| `string` | 字符串 | `铁剑` |
| `bool` | 布尔值 | `true`、`false`、`1`、`0` |
| `array_int` | 整数数组，使用 `[ , , ]` 格式 | `[101, 102, 103]` |
| `array_float` | 浮点数组，使用 `[ , , ]` 格式 | `[1.5, 2.0, 3.5]` |
| `array_string` | 字符串数组，使用 `[ , , ]` 格式 | `[火球, 冰冻, 雷击]` |

### Sheet 忽略规则

以下 Sheet 不会导出：

- 名称以 `#` 开头
- 名称以 `_` 开头

这类 Sheet 可用于备注、草稿或废弃数据。

### Sheet 命名规则

- `Sheet` 名称就是导出的 JSON 文件名
- 例如 `Sheet` 名叫 `Item`，导出后会生成 `Output/Json/Item.json`
- 同时也会生成对应的 `ItemConfig.cs`
- 因此 `Sheet` 名不要随意修改，尤其不要在未通知程序同学的情况下改名

如果修改了 `Sheet` 名，可能产生以下影响：

- 导出的 JSON 文件名会一起改变
- 导出的 C# 类名也会一起改变
- 程序侧如果仍按旧名称读取配置，可能出现找不到配置文件或加载失败

如果出现重名 `Sheet`，请特别注意：

- 不同 Excel 文件中，如果有两个或多个 `Sheet` 同名，最终只会保留一份同名导出结果
- 后处理到的同名 `Sheet` 会覆盖前面的结果
- 这会导致前一份同名配置被悄悄顶掉，容易造成数据混乱

建议：

- 每张正式配置表都使用唯一的 `Sheet` 名
- 不要把不同含义的配置表命名成同一个 `Sheet` 名
- 若确实需要改名，请提前和程序同学确认

## 6. 导出结果说明

### 6.1 JSON 文件

- 每个有效 Sheet 会生成一个同名 JSON 文件
- 文件位于 `Output/Json/`
- 外层固定带 `items` 数组

示例：

```json
{
  "items": [
    {
      "id": 1001,
      "name": "铁剑",
      "attack": 15.5,
      "skill_ids": [101, 102]
    }
  ]
}
```

### 6.2 C# 文件

- 每个有效 Sheet 会生成一个 `xxxConfig.cs`
- 文件位于 `Output/Scripts/`

示例：

```csharp
[System.Serializable]
public class ItemConfig
{
    public int id;
    public string name;
    public float attack;
    public int[] skill_ids;
}
```

### 6.3 manifest.json

- 记录每张表对应的 Excel 文件、JSON 文件和 C# 类名
- 供程序侧统一加载配置时使用

### 6.4 ConfigTables.cs

- 记录全部配置表名的枚举
- 供程序侧做统一索引和管理

## 7. 报错怎么看

如果导表失败，控制台会打印中文错误信息。

错误格式如下：

```text
[错误] 文件名 - Sheet '表名' - 第 行号 行 '列名' 列：错误原因
```

常见报错示例：

```text
[错误] Item.xlsx - Sheet 'Item' - 第 5 行 'attack' 列：期望 float 类型，实际值为 '很高'
[错误] Store.xlsx - Sheet 'Store'：缺少必填主键字段 'id'
[错误] Skill.xlsx - Sheet 'Skill'：id 值 101 重复出现
```

重要说明：

- 只要有任意错误，工具就不会生成有效输出
- 请根据报错修改 Excel 后重新导表

## 8. 常见问题

### 8.1 为什么双击后提示找不到 Excel 文件？

原因：

- 没有创建 `Excel` 文件夹
- Excel 文件没有放在 `Excel` 文件夹中
- 文件格式不是 `.xlsx`

### 8.2 为什么某个 Sheet 没有导出？

请检查：

- Sheet 名是否以 `#` 或 `_` 开头
- Sheet 是否至少有 4 行
- 是否有实际数据行
- 第 1 行和第 3 行中，是否有一行正确填写了类型

### 8.3 为什么没有生成任何输出？

原因通常是导表过程中存在错误。

请查看控制台中的 `[错误]` 信息，修复后重新运行。

### 8.4 可以一个 Excel 里放多个 Sheet 吗？

可以。工具会逐个处理每个有效 Sheet。

### 8.5 数组怎么填？

数组字段请使用中括号和逗号 `[val1, val2, val3]` 的格式，例如：

```text
[101, 102, 103]
[战士, 法师, 游侠]
[1.5, 2.0, 3.5]
```

## 9. 交付给程序的内容

导表完成后，通常把以下内容交给程序同学：

- `Output/Json/` 下全部 JSON 文件
- `Output/Scripts/` 下全部 C# 文件
- `Output/manifest.json`
- `Output/ConfigTables.cs`

---

## 10. 推荐工作习惯

- 每次导表前先关闭相关 Excel 文件
- 字段名一旦确定，尽量不要频繁修改
- 新增字段时同时确认类型填写正确
- 导表成功后再把 `Output` 内容提交给程序
- 如果报错，优先按控制台提示定位到具体文件、Sheet、行和列
